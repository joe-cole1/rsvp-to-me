Design notes
• Single entry point: one "+ Add section" button. No preset type chips.
• Icon drives identity: selecting an icon auto-fills the title (overridable).
• No type lock-in: any section can have any icon; duplicates allowed.
• URL always available: link field shown on all sections, not just LINK type.
• Lucide icons in prod: emoji here, actual Lucide SVGs in the real component.
• DB change minimal: store iconKey string on the InfoSection row; existing type column can be repurposed or left for backward compat.
